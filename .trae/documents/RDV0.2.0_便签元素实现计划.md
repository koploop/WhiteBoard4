# RDV0.2.0 便签元素实现计划

## 1. 目标与范围
- 目标：实现 RequestDoc 中 RDV0.2.0 的“插入便签元素”完整能力，覆盖插入、选中、编辑、批注、擦除、缩放自适配与画布联动。
- 范围：
-  - 新增便签元素（基于原生 View 能力）并接入白板元素体系。
  - 抽象通用原生 View 承载层，为后续 WebView/自定义 View 元素复用。
  - 接入 MoreToolsPanel 的“插入便签”入口。
  - 扩展选中能力与选中菜单动作，支持文字编辑、批注、便签内擦除。
  - 保持与现有并行状态机、无限画布、选中处理链路兼容。
- 不在本次范围, 但是需要考虑后续扩展：
  - 容器类元素体系（RDVxxxx）的通用化重构。
  - 新增外部存储持久化格式。

## 2. 现状结论（基于当前代码）
- `ToolData` 已存在“便签”工具项，但 `MainActivity` 点击分支未实现，当前仅有图片接入。
- 元素能力体系已具备基础能力（移动、缩放、旋转、颜色、层级等），但缺少“文字编辑/批注/便签内擦除”能力定义与菜单动作。
- 擦除链路已存在（状态机 -> SDK -> 元素 `erase`），但“便签内批注线条擦除”尚无实现。
- 当前未发现 `NoteElement` 实现类与便签 UI 载体。

## 3. 实施步骤（按执行顺序）

### 步骤1：补齐便签能力定义与动作协议
- 修改 `boardelement` 中选中能力枚举，新增便签所需能力：
  - 文字编辑能力
  - 批注能力
  - 便签内擦除能力
- 修改 `boardstate` 的 `SelectedHandler`：
  - 菜单构建逻辑按能力显示新按钮。
  - 点击事件分发新增动作类型（进入编辑、切换批注、切换便签内擦除）。
- 修改 `boardinterface`/`boardimpl` 的接口与实现：
  - 增加便签编辑状态切换、便签批注模式切换相关 API（仅便签目标生效）。

### 步骤2：实现 NoteElement（元素模型与绘制）
- 在 `boardelement` 新增 `NoteElement`：
  - 继承现有元素基类，保留统一变换与选中回调机制。
  - 内部维护：
    - 文本内容与文本样式
    - 便签背景色
    - 便签内部批注笔迹集合（独立于全局元素列表）
    - 编辑态/批注态/便签内擦除态
  - 初始化时注册能力：移动、缩放、改变颜色、文字编辑、批注、擦除。
- 实现 `draw`：
  - 先绘制便签底色与文本容器区域，再绘制批注层，保证“线条在文字下层”的视觉要求。
- 实现命中与边界：
  - 命中以便签外接矩形为准，适配当前元素变换矩阵。
- 实现 `erase`：
  - 仅作用于便签内部批注线条，不影响文字内容本体。

### 步骤3：实现通用原生 View 承载层与画布变换同步
- 在 `boardimpl`（SurfaceView 所在模块）新增通用 `NativeViewHostLayer`（命名可按现有规范微调）：
  - 定义原生 View 元素契约（如：创建 View、绑定数据、更新变换、销毁回收）。
  - 统一维护“元素ID -> 原生View实例”映射，避免在具体元素里分散管理。
  - 提供生命周期入口：元素新增、元素更新、元素删除、画布变换更新。
- 在承载层统一完成矩阵同步：
  - 在每帧或状态变更时将元素世界坐标与画布矩阵换算为屏幕坐标。
  - 同步原生 View 的位置、缩放、旋转，保证与白板元素模型一致。
- 便签作为首个实现：
  - `NoteElement` 通过承载层创建并管理 `EditText/TextView`。
  - 编辑态焦点与键盘行为在承载层中统一处理（请求焦点、拉起/收起键盘、文本回写）。
- 扩展约束：
  - 承载层 API 不包含便签语义字段，后续可直接接入 WebView、任意自定义 View 元素。
  - 选中框命中基于元素模型计算，保证不同原生 View 类型行为一致。

### 步骤4：接入“插入便签”入口
- 修改 `app` 的 `MainActivity`：
  - 在 MoreToolsPanel 的 `note` 点击分支创建并插入 `NoteElement`。
  - 插入策略与图片一致：基于当前视口居中放置、给默认尺寸与默认背景色。
- 点击后自动关闭面板并选中新建便签，提升可操作性。

### 步骤5：实现便签专属交互状态切换
- 在现有状态机链路中补齐便签专属操作入口：
  - 选中便签后点击“文字编辑”进入编辑态。
  - 点击“批注”进入批注态后，触摸写入便签内部批注线条集合。
  - 点击“擦除”进入便签内擦除态后，仅擦除便签内部批注线条。
- 非便签元素屏蔽上述专属动作，避免污染通用元素行为。

### 步骤6：实现缩放时内容自适配与滚动
- 便签缩放后触发文本容器重排：
  - 依据新宽高更新文本显示区域与字体排版。
- 当文本超出显示区域时：
  - 启用垂直滚动手势查看全文，不改变便签外部尺寸。
- 保证缩放过程中文本不丢失、光标与选择行为稳定。

### 步骤7：兼容撤销重做与复制粘贴
- 为 `NoteElement` 实现深拷贝：
  - 复制文本、背景色、变换参数、批注线条数据与能力集合。
- 接入现有 undo/redo 快照链路：
  - 插入、删除便签可撤销重做,便签内的编辑文字、批注新增/擦除暂不支持撤销重做。

### 步骤8：文档同步更新
- 更新 `docs/WhiteBoard.md`：
  - 增补 NoteElement 结构、通用 NativeViewHostLayer 设计与状态切换设计。
  - 说明便签内批注/擦除的数据归属与渲染顺序。
- 更新 `docs/CHANGELOG.md`：
  - 记录 RDV0.2.0 新增点、变更点、修复点。
- 更新 `docs/COMMUNICATION.md`：
  - 追加本轮用户输入记录。

## 4. 验证计划
- 构建验证：
  - 执行项目编译，确保各模块接口变更一致、无编译错误。
- 功能验证（手工）：
  - 插入便签：从更多工具点击后可见、默认居中、可选中。
  - 选中能力：移动/缩放/改色/文字编辑/批注/擦除按钮可见且可用。
  - 编辑行为：进入编辑态弹键盘；退出后文本保留。
  - 批注行为：便签内可画线，线条位于文字下层。
  - 擦除行为：仅擦除便签批注线条，不擦除文字。
  - 画布联动：画布缩放/平移时便签与批注同步。
  - 缩放排版：缩放后文本重排，超出可滚动查看。
  - 撤销重做：插入、编辑、批注、擦除均可回退与重做。

## 5. 风险与应对
- 风险1：SurfaceView 与原生子 View 叠加坐标错位。
  - 应对：统一通过世界坐标 -> 屏幕坐标换算，集中管理矩阵同步。
- 风险1.1：承载层与具体元素耦合后影响后续扩展。
  - 应对：先定义最小通用契约，便签仅作为契约实现方，不向承载层注入业务分支判断。
- 风险2：编辑态触摸事件与画布状态机冲突。
  - 应对：编辑态下由便签 View 优先消费事件，退出编辑态恢复状态机分发。
- 风险3：便签内批注与全局擦除链路耦合过高。
  - 应对：便签内批注数据独立管理，仅通过 `NoteElement.erase` 暴露擦除入口。
